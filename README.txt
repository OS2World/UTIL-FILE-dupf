インストール

    アーカイブには、フルソースコードとバイナリ、マニュアル、および、dupf
    のフロントエンドとして使っているREXXスクリプトが含まれています。
    
    dupfのバイナリをそのまま使う場合には、
    
        dupf.exe    実行モジュール

    をPATHで指示されるディレクトリに置いてください。使い方については
    
        dupf.man

    を御覧ください。
    
    フロントエンド用のREXXスクリプトは
    
        lstdups.cmd     重複ファイルの一覧
	deldups.cmd     重複ファイルの削除

    の二つです。これらは、私の手元の環境に合わせてありますので、異なった
    環境で使用する場合には多少の修正が必要です。

    なお、dupf.exe はＥＭＸ／ＧＣＣで作成されていますので、実行には
    ＥＭＸのランタイム（0.9c以上）が必要です。これは別途入手してセット
    アップしておいてください。

dupf.exe - 重複ファイルの検索

    これは標準入力からファイル名（パス名）の一覧を受け取り、その中で重複
    しているものをリストアップします。詳しくは、dupf.manを御覧ください。
    
    私は、GNU findと組み合わせて、
    
        find 検索パス ... -print | dupf

    として使っています（下記のREXXスクリプトを参照）。

lstdups.cmd - 重複ファイルの一覧

    これはパラメタで指示された検索パスの下にあるファイルについて、重複ファ
    イルをリストアップします。

        lstdups 検索パス ... 

    このスクリプトを使うにはdupfの他にGNUのfind（あるいは同等の機能を持っ
    たプログラム）が必要です。

deldups.cmd - 重複ファイルの削除

    これは、重複ファイルを（ユーザの指示に基づいて）削除します。使い方は、
    lstdups と同様に検索パスを指示する方法
    
        deldups 検索パス ...

    あるいは、lstdups の出力結果（重複リスト）を渡す
    
        deldups -f 重複リストファイル

    とがあります。
    
    このスクリプトを使うには、GNUのfind、cp、およびテキスト（ＶＩＯ）モー
    ドのエディタが必要です。それぞれ、スクリプト中の
    
        find = 'find'       GNU find
        copy = 'cp'         GNU cp
        edit = 'me'         VIO ベースのエディタ
    
    で設定して下さい。またこのスクリプトは一時ファイルを作成します。一時
    ファイルのパスが「D:\tmp」で固定になっていますので、適時変更しておい
    てください。

    スクリプトを起動すると、重複リストがエディタで表示されます。
    
        検索パス指定の場合には相当時間がかかります。最初のうちは、あらか
        じめ重複リストを作っておいた方が無難でしょう。

    同一内容のファイルが

    ---
        file1
	file2
	file3
    ---

    といった形式で表示されます。削除したいファイルについて

    ---
    d   file1
        file2
    d   file3
    ---

    と行の先頭に'd'でマークします。ファイルを更新してエディタを終了すれば、
    マークされたファイルが削除されます。
